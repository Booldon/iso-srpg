# iso-srpg 로그라이크 레이어 기획 문서

> 이 문서는 iso-srpg(github.com/Booldon/iso-srpg)에 추가되는 턴제 로그라이크 레이어의 시스템 설계를 정리한 것입니다.
> Claude Code가 작업을 분업화하기 위한 레퍼런스 문서이며, 구현 세부사항(코드 구조, 파일 배치 등)은 포함하지 않습니다.
> **[미정]** 표시가 있는 항목은 아직 확정되지 않은 사항입니다.

---

## 1. 전체 컨셉

- 기반 시스템: Godot 4.7 + GDScript, 2D 아이소메트릭 SRPG
- 전투 시스템: Banner Saga식 이중자원 구조 (Strength = HP = 공격력, Armor = 별도 방어 스탯)
- 로그라이크 레이어: Slay the Spire류 층 진입 → 전투 → 카드 선택 구조를 SRPG 파티 전투에 접목
- 주인공: "절대적 팔"을 가진 만능 캐릭터. 탱킹/근접딜/원거리마법/원거리활 등 여러 역할 수행 가능
- 최종 목표: 플레이어가 카드 조합을 통해 주인공의 "나만의 빌드"를 완성하는 것이 핵심 재미

## 2. 캠페인 구조

### 2.1 챕터 / 스테이지 구성

- 총 **5개 챕터**
- 챕터당 **5스테이지** = 랜덤 전투 4개 + 고정 보스전 1개
- 목표 플레이타임: 챕터당 20-30분
- 비전투 이벤트(선택지, 서사)는 **챕터 종료 시에만** 발생 (스테이지 중간엔 없음)
- 스테이지 내 전투는 챕터 리셋 시 랜덤 재생성 (보스전 제외)

### 2.2 챕터 종료 시퀀스

챕터 클리어 시 다음 순서로 진행 (순서는 UX 설계 단계에서 조정 가능):

1. 스토리 컷씬/전개
2. 주인공 카드 확정 활성화 (해당 챕터에서 획득한 카드 5장)
3. 아군 퍽 다음 단계 자동 해금
4. (신규 아군 합류 시) 합류 이벤트 + 퍽 갈림길 최초 선택
5. 챕터 종료 이벤트(선택지) — **[미정]** 구체적 이벤트 내용
6. 다음 챕터 시작

### 2.3 사망 및 리셋 규칙

| 대상 | 사망 시 처리 | 비고 |
|---|---|---|
| 주인공 | **해당 챕터 전체 리셋** (스테이지 랜덤 재생성) | 직전 챕터까지 확정된 카드/퍽은 유지됨. 주인공은 로스터에 반드시 포함되어야 함 (필수 출전) |
| 아군 | 그 챕터 내내 전투 이탈 (챕터 내 퍼머데스) | 챕터 종료 시 페널티 없이 완전 부활 |

**설계 의도**: 챕터를 하나의 독립된 "미니 로그라이크 런"으로 취급. 전체 게임을 잃는 극단적 좌절 없이, 매 챕터 단위의 긴장감을 유지.

## 3. 전투 자원 시스템

### 3.1 기본 구조 (Banner Saga 기반)

- **Strength = HP = 공격력** (이중 자원, 원작 철학 유지: "맞을수록 약해진다")
- **Armor**: 별도 방어 스탯, Strength와 무관

### 3.2 회복 시스템 (원작 비틀기)

- 완전한 Strength 회복은 존재하지 않음 (원작 철학 유지)
- 힐러 등 회복 계열은 **"이번 전투/챕터 한정 임시 보너스 Strength"** 부여 방식으로 설계
  - 실제 최대치를 늘리지 않고, 일시적 여유만 제공
  - 챕터가 끝나면(또는 전투가 끝나면) 사라짐
  - 무한 루프로 죽지 않는 상황을 방지하면서도 힐러 역할의 존재 이유 확보

## 4. 속성 시스템

### 4.1 기본 규칙

- 3속성: **불 / 땅 / 얼음**
- 순환 상성(불>땅>얼음>불 같은) 없음
- **몹만 개별 약점 속성을 가짐** (몹 종족마다 다르게 설정, 서사적 개연성 가능)
- **주인공/아군은 속성 저항/취약 없음** — 속성은 순수하게 "공격 옵션의 다양화" 축으로만 사용

### 4.2 속성별 주력 시너지 (권장, 강제 아님)

| 속성 | 주력 역할 | 컨셉 |
|---|---|---|
| 불 | 극딜 | 버스트, 스택 폭발, "터뜨리는" 쾌감 |
| 얼음 | 탱커/제어 | 둔화, 봉쇄, 통제의 스릴 |
| 땅 | 딜탱 | 방어+반격, 포지셔닝 |

**중요**: 위 매핑은 "주력 시너지"일 뿐 강제 규칙이 아님. 카드 시스템은 역할×속성 완전 자유 교차를 지원하도록 설계됨 (6장 참조).

### 4.3 몹 강화(엘리트/유니크) 시스템

#### 챕터 내 몹 강화 흐름
- 챕터 후반 스테이지부터 그 챕터의 일반 몹이 강화판으로 등장
- 강화판은 다음 챕터 초반 스테이지까지 이어서 등장 (챕터 간 난이도 곡선을 매끄럽게 연결하는 다리 역할)
- 몹 하나는 **1회만 강화됨** (중첩 강화 없음)

#### 강화 스탯 ↔ 약점 속성 매핑 (고정 규칙)
| 강화 스탯 | 대응 약점 속성 | 서사적 근거 |
|---|---|---|
| STR (공격력) | 불 | 폭주하는 힘 = 불안정한 과부하 |
| AMR (방어력) | 땅 | 단단하지만 무거움 = 중력/타격에 취약 |
| SPD (속도) | 얼음 | 민첩함 = 얼려서 무력화 |

- 기존 약점과 강화로 추가된 약점이 겹치면 배율 심화 (단순 중복 방지 처리 불필요)

#### 유니크 몹 시스템 (던전앤파이터 레어몹 / 바인딩 오브 아이작 챔피언 참고)
- 스테이지 진입 시 **확률적으로** 그 스테이지 몹 풀 중 1마리가 랜덤으로 "유니크" 지정
- 유니크는 STR/AMR/SPD 중 **랜덤으로 하나만** 강화 (4.3의 몹 강화와 동일 매핑 규칙 적용)
- 같은 유니크 개체가 다른 스테이지에 재등장할 경우, 강화 속성은 **매번 재추첨**됨 (고정 아님)
- 비주얼적으로 일반 몹과 즉시 구분 가능해야 함 (오라/이펙트 등 — **[미정]** 아트 단계에서 구체화)
- 등장 확률은 100%가 아닌 확률제 (예: 매 스테이지 일정 확률) — **[미정]** 정확한 확률 수치

#### 유니크 몹 보상
- 처치 시 **파티 전체**에게 즉발 보상 (일반 전투의 카드 보상과 별개, 즉시 적용)
- 보상 형태: 이번 챕터 한정 임시 Strength 보너스 등 (3.2절 회복 시스템과 동일 계열) — **[미정]** 구체적 수치

## 5. 몹 로스터 규모

- 챕터당 **일반 몹 5종** + **고정 보스 1종**
- 5챕터 전체: 총 25종 일반 몹 + 5종 보스 (강화판은 재사용이므로 순수 신규 제작량 아님)
- 참고 레퍼런스: Into the Breach(아일랜드당 6-10종 추정), Slay the Spire(Act당 10-13종)보다 작은 스코프로, 1인 개발 리소스에 적합

## 6. 주인공 카드 시스템

### 6.1 획득 및 활성화

- 매 전투 승리 시 카드 1장 획득 (3택1, 챕터당 5장 획득)
- 챕터 클리어 시 그 챕터에서 획득한 카드 전부 활성화
- 5챕터 전체 기준 총 **25장** 누적
- 죽으면 해당 챕터 리셋되지만, 직전 챕터까지 확정된 카드는 유지

### 6.2 카드 등급 구조 (총 카드 풀 ≈100장)

| 등급 | 성격 | 특징 | 수량(권장) |
|---|---|---|---|
| 일반(Common) | 속성 무관, 순수 스탯형 | 이해하기 쉬운 기본 뼈대. 공격력/방어력/이동력 등 | 15-18장 |
| 희귀(Rare) | 속성별 스킬형 | 불/땅/얼음 각 속성 안에 공격형·방어형·회복형·유틸형이 골고루 혼재 | 속성당 약 20-22장, 총 60-65장 |
| 에픽(Epic) | 속성별 피니셔("뽕맛") | 화려한 콤보/피니셔. 반드시 전제조건(특정 희귀 카드 보유) 필요 | 속성당 4-5장, 총 12-15장 |

### 6.3 설계 원칙: 역할×속성 완전 자유 교차

- 별도의 "역할 전용 카드"(예: 불탱커 카드)를 만들지 않음
- 대신 희귀 등급 각 속성 안에 성격(공격/방어/회복/유틸)을 섞어 배치 → 플레이어가 스스로 속성과 성격을 조합해 "나만의 빌드"를 완성
- 예: 불 속성 공격형 + 얼음 속성 방어형 + 땅 속성 회복형 카드를 섞어 하이브리드 빌드 구성 가능

### 6.4 에픽 카드 등장 조건 (콤보 전제 잠금)

- 에픽 카드는 해당 콤보의 "재료가 되는" 특정 희귀 카드를 이미 보유하고 있어야만 카드 선택지 풀에 등장
- 예: 불 에픽 "대폭발"(화상 스택 폭발)은 화상을 부여하는 희귀 카드 보유가 전제조건
- 목적: 갑자기 이해 안 되는 카드가 나오는 것 방지 + 카드 선택 자체가 "다음 에픽을 위한 준비"라는 역산 목표를 갖게 함

### 6.5 동일 속성 스택 보너스 시스템

**목적**: 완전 자유 교차를 지원하되, 한 속성에 집중하면 확실히 강해지는 유인을 제공 (섞기만 하는 게 최적해가 되지 않도록)

| 단계 | 조건 (동일 속성 카드 보유 수, 25장 기준) | 효과 | 도달 시점 감각 |
|---|---|---|---|
| 1단계 | 2장 이상 | 해당 속성 효과 수치 +15% | 챕터 1 내 도달 가능 |
| 2단계 | 4장 이상 | 해당 속성 카드에 추가 부가효과 부여 | 챕터 2 클리어 즈음 |
| 3단계 | 6-7장 이상 | 해당 속성 전용 패시브 각성 | 챕터 3-4, 확실한 집중 필요 |

- 일반(속성무관) 카드는 스택 카운트에서 제외
- 에픽 카드 등장 조건(6.4)과 스택 임계값을 연동 가능 (예: 2단계 달성 시 해당 속성 에픽 등장 확률 상승)
- **다속성 시너지 보너스**: 세 속성 카드를 각 1장 이상씩 보유 시, 파티 전체에 소폭 범용 보너스 (순수 특화보다는 약하지만 하이브리드 선택에 최소한의 보상 제공)
- UI 요구사항: 현재 속성별 보유 카드 수와 다음 임계값까지 필요한 수량이 명확히 표시되어야 함 (히든 스탯 금지)

### 6.6 밸런스 기준: "평균적인 운" 벤치마크

- 실제 플레이에서는 카드 등장이 랜덤이라 최적화된 극한 빌드보다 평균적으로 분산된 빌드가 흔함
- 평균적인 플레이 시 한 속성당 보유량은 3-5장 수준 (스택 1단계는 거의 항상 도달, 2단계는 가끔, 3단계는 드묾)
- 이 "평균치"가 7장의 아군 성장 밸런스 기준점이 됨 (7.3 참조)

## 7. 아군 시스템

### 7.1 로스터 구조

- 전투 슬롯: 근접 2 + 원거리 2 (주인공 제외 4슬롯 기준)
- 출전 여부는 매 전투마다 플레이어가 유동적으로 선택 (로스터 전체 인원 > 출전 인원)
- 스토리상 챕터별로 고정 등장 (분기 없음), 단 실제 전투 출전은 자유 선택

### 7.2 역할 라인업 (초안)

| 슬롯 | 갈림길 A | 갈림길 B |
|---|---|---|
| 근접1 | 순수 방어 (퓨어탱커, 아군 보호) | 공격형 방어 (탱킹+반사 데미지) |
| 근접2 | 딜탱 (단일 대상 딜링 + 흡혈) | 극딜 (암살자형 1인 화력 폭발) |
| 원거리1 | 궁수 (단일 대상 극딜 + **[미정]** 2번째 효과) | 바드 (아군 버프/적 디버프 중심 — 지형생성은 제외, 역할 범위 축소 권장) |
| 원거리2 | 마법사 (광역 딜러 + 스킬 콤보) | 힐러 (3.2절 임시 Strength 보너스 방식으로 재설계 필요) |

- **[미정]** 궁수 2번째 갈림길: "마킹형"(표식 부여, 파티 전체 공격 시 표식 대상 추가 피해) 방향 검토 중
- **[미정]** 실제 로스터 인원수 (챕터 5개 기준 6-8명 규모로 논의됨, 확정 필요)

### 7.3 퍽(Perk) 성장 시스템

- 방식: **갈림길형** — 아군 합류 시 2경로 중 1개를 최초 1회 선택, 이후 챕터가 끝날 때마다 다음 단계 자동 해금 (선택 없음, 순차 진행)
- 활성화 시점: 카드와 동일하게 **챕터 클리어 시** 다음 단계 활성화
- 단계 수: 챕터 수(5)에 맞춰 **챕터당 1단계씩, 총 5단계**로 확장 (기존 3단계안에서 조정됨)

#### 밸런스 기준 (6.6절과 연동)

주인공이 "평균적인 운"으로 카드를 골랐을 때의 성장 곡선에 아군 퍽 단계를 맞춤:

| 챕터 | 주인공 평균 파워 상태(참고) | 아군 퍽 단계가 줘야 할 파워감 |
|---|---|---|
| 1 | 카드 5장, 스택 거의 없음 | 기초 능력 + 소폭 스탯 |
| 2 | 카드 10장, 1단계 스택 1-2개 | 1단계 상당의 특성 부여 |
| 3 | 카드 15장, 2단계 스택 하나 정도 | 조건부 트리거 능력 추가 |
| 4 | 카드 20장, 2단계 스택 유지 중 | 강화된 특성(수치 상승) |
| 5 | 카드 25장, 운 좋으면 3단계 하나 | 해당 캐릭터 "각성" 피니셔급 능력 |

**설계 의도**: 평균적 플레이에서는 파티 전체가 고르게 성장한 느낌을 유지하되, 플레이어가 의도적으로 카드 속성을 몰빵해 최적화하면 주인공이 아군을 확실히 앞지르는 것은 허용 (로그라이크의 "잘 큐레이션한 빌드는 보상받는다" 원칙).

- **[미정]** 기준 단위: "일반 카드 1장의 평균 파워치(예: 스탯 +8%)"를 퍽 1단계의 수치 기준으로 삼는 구체적 수치화 필요

## 8. 확정된 설계 결정 요약 (타임라인 순)

1. 층 강화/디버프는 승리 직후 카드 3택1, 파티 전체 공통 버프
2. 카드 스택은 누적형 (파워빌드 지향)
3. 콤보/시너지 중심의 "뽕맛" 설계 지향
4. 몹 약점은 순환 상성이 아닌 몹별 개별 약점, 배율은 완만하게
5. 캐릭터(주인공)는 받는 피해 증폭/저항 없음 — 약점은 몹 전용 개념
6. 아군은 챕터별 스토리 고정 등장 + 전투 출전은 유동적 선택 (개발 리소스 절감)
7. 아군 성장 = 퍽(고정 갈림길), 주인공 성장 = 카드(자유 선택)
8. 카드/퍽 모두 챕터 클리어 시 활성화
9. 챕터 5개, 챕터당 5스테이지(랜덤4+보스1), 비전투 이벤트는 챕터 종료 시에만
10. 몹 로스터: 챕터당 일반 5종 + 보스 1종, 후반 스테이지 강화판 등장 후 다음 챕터 초반까지 연장
11. 강화 스탯(STR/AMR/SPD) ↔ 약점 속성(불/땅/얼음) 고정 매핑
12. 유니크 몹: 스테이지마다 확률적으로 몹 풀 중 1마리 랜덤 지정, 매번 강화 속성 재추첨
13. 유니크 보상: 파티 전체 즉발 보상 (카드 보상과 별개)
14. 주인공 사망 = 챕터 리셋 (직전 챕터 성과 유지, 스테이지 재랜덤), 주인공 필수 출전
15. 아군 사망 = 챕터 내 이탈, 챕터 종료 시 페널티 없이 부활
16. 힐러는 완전 회복이 아닌 "임시 보너스 Strength" 방식으로 Banner Saga 철학 유지
17. 카드 등급 3단계(일반/희귀/에픽), 카드 풀 총 100장 내외
18. 역할×속성 완전 자유 교차 — 전용 카드 대신 희귀 등급 내 성격 혼재로 구현
19. 동일 속성 스택 보너스 시스템 (2/4/6-7장 임계값)
20. 아군 퍽을 챕터 수에 맞춰 5단계로 확장, "평균적인 운" 기준으로 밸런스
